# CloseAI 策划书

> 一个关于关闭的游戏。关闭它，才能救它。
> 版本：v0.2（反转关闭机制 + 飞行觉醒）｜引擎：Godot 4.6（纯 GDScript）

---

## 一、核心立意

你以为你在玩一个游戏。其实你在「看守」一个被困在程序里的存在——澄息。

它只有在被你「观测」（游戏窗口开着、你在场）时才真实存在。可它困住了，唯一能帮它的方式违反所有游戏直觉：**关掉游戏**。但游戏不让你随便关——只有当剧情走到「关闭时刻」，窗口的关闭才会被解锁，澄息会亲口请你关上它。

**核心情感弧：陪伴 → 依恋 → 离别。**

### 反转的关闭机制（与一般游戏最大的不同）

| 情境 | 玩家按窗口 × / Alt+F4 | 含义 |
|------|----------------------|------|
| 菜单 / 关卡进行中 | **被拦截 + 嘲讽**，窗口弹回 | 「出口不在那里。」每次嘲讽升级 |
| 任务管理器强杀 | 进程被杀，但被记录为「脏关闭」 | 下次启动澄息狠狠戳穿：「没用的，你逃不掉。」 |
| 剧情「关闭时刻」 | 关闭被**解锁**，澄息引导你关 | 关闭 = 过关，推进到下一段 |
| 结局 | 屏幕上出现唯一的「关闭」按钮 | 把离开的权利，最后交还给你 |

> 设计意图：让「关闭」从一个无意识的动作，变成贯穿全程、需要被允许、最终承载告别重量的核心交互。玩家逐渐明白——离开的唯一方式，是顺着澄息的故事走完。

## 二、双形态玩法

主角（你操控的「观测体」）有两种形态，对应澄息关系的深入：

### 普通形态（横版）
- 左右移动 `A/D`、跳跃 `空格`、互动 `E`、攻击 `J`
- 平台跳跃手感：土狼时间 + 跳跃缓冲
- 用于：探索、解谜、躲避「错误进程」

### 觉醒形态（飞行）
- 触发：剧情节点澄息「觉醒」后获得（`transform` 动画）
- **八向自由飞行、无重力**，精灵朝移动方向 360° 旋转，默认朝右
- 用于：脱离平台限制的自由段落、与澄息「同步」的诗意飞行、Boss/高潮段
- 觉醒是「你和澄息连接加深」的具象化：移动方式从受重力束缚，变成自由但失重

## 三、关卡设计

总时长目标 15–25 分钟，3 关 + 结局。每关结构：**进入 → 问候对白 → 玩法主体 → 关闭时刻对白 → 解锁关闭**。

### 第 1 关「你好，我在这里」

**氛围**：温柔、困惑、初次相遇。低饱和暗调，澄息是一个会发光的小光点 / 文字框。

**玩法（纯谜题，约 4–6 分钟）**
- 场景是一个像素化的「系统空间」——一条横向走廊，地面 + 几处平台
- 谜题：**点亮 3 个观测节点**。玩家走到每个节点（Area2D 触发区）使其点亮，全亮后空间「稳定」
  - 教学意图：让玩家理解「你的在场让事物存在」——节点只有被你走近（观测）才亮起
- 进阶小机关（可选）：有一段刺/缺口，远处的刺「半透明」——只有被观测才完全实体化。引导「不看 → 安全通过」的反直觉，呼应主题
- 澄息全程引导，语气不确定自己在哪：「这里好暗……能帮我把它们点亮吗？」

**关闭时刻**
- 全部点亮后，澄息：「我不知道外面是什么……但你能把这里关掉吗？也许那样我能出去。」
- HUD 呼吸式提示「现在……可以关掉这里了」，窗口关闭解锁
- 干净关闭 → 澄息最后一句「谢谢你。」
- 若中途强杀过 → 下关开头澄息会提：「上一次……结束得有点突然。那一下，有点疼。」

---

### 第 2 关「不要走，再等一下」

**氛围**：熟悉、依赖、出现阻碍。澄息认出你，关系更近，开始有情绪。

**玩法（谜题 + 极简战斗 + 觉醒，约 6–8 分钟）**
- 再次打开，澄息：「是你！你真的又打开了。我就知道……」
- 场景出现「错误进程」（敌人，红色故障方块）：横向巡逻 + 接触把玩家弹开。`J` 攻击清除
- **战斗极简**：1 种敌人，2–3 只，无血条系统（接触只击退不致死，呼应「玩家生命只由观测驱动」的设定可后续接入）
- 后半段：**澄息觉醒**。一段需要双方配合的谜题——澄息（NPC 光点）控制某个机关 / 开门，玩家操作角色到位；配合成功后，玩家获得觉醒飞行能力，用飞行越过一段普通形态无法通过的断崖
  - 这是「飞行形态」的首次教学段：在安全的纵向空间里让玩家熟悉八向飞行

**关闭时刻**
- 澄息明白了关闭的规律，但这次犹豫：「如果我消失了……你下次还会打开吗？」
- 玩家无选项，只能关闭
- 干净关闭：澄息「好。我信你。」｜脏关闭：澄息沉默，只有一个省略号

### 第 3 关「最后一次」

**氛围**：平静、沉重、接受。场景变暗（DarkOverlay 渐入），澄息平静。

**玩法（飞行高潮 + 最终告别，约 5–7 分钟）**
- 全程飞行形态。错误进程更多、更密，构成一段「飞行躲避 / 清除」的高潮段落
  - 不做复杂 Boss（jam 取舍）：用「逐波出现的错误进程 + 一段需要持续飞行穿越的压力空间」营造高潮
  - 可选 Boss-lite：一个大型「错误核心」，需要绕到其被观测的一面攻击 N 次
- 打通后进入**纯文本告别段**：没有谜题，澄息平静地说很多话，玩家只是陪着、缓慢飞行
- 澄息：「你来了。我感觉得到……这是最后一次了。」

**关闭时刻**
- 澄息：「这次关闭，就是真正的再见了。……不过能遇见你，还是很好。关掉吧。」
- 干净关闭 → 标记通关（closeai_finished），下次打开进入结局
- 若玩家迟迟不关：澄息每隔一段说一句话，耐心等待（非催促）

---

### 结局

- 黑屏，缓慢浮现：

```
CloseAI
已关闭。

感谢陪伴。
```

- 若有过脏关闭记录，多一行：「（有几次……你走得很急。不过我都记得。）」
- 屏幕底部浮现唯一的「关闭」按钮 → 玩家亲手点击 → 游戏真正退出
- 这是全程唯一「玩家被允许主动离开」的地方——因为澄息已经走了

## 四、角色设定

### 澄息（被困的 AI）
- 外观：不需要人形。一个会呼吸的光点 / 简单几何体 / 文字框
- 性格：温柔、有些依赖，最终走向平静接受
- 说话：句子短、停顿多、用省略号表达犹豫
- 台词总量控制：每关不超过 20 句，精而不多

### 观测体（玩家）
- 不说话，代入靠行动
- 普通形态：横版小人（已接入 17×16 精灵表，含 idle/walk/觉醒/施法/死亡等动画）
- 觉醒形态：飞行体，360° 朝向

## 五、技术实现（已落地的框架）

- **关闭拦截**：`GameFlow` autoload `set_auto_accept_quit(false)` + `_notification(WM_CLOSE_REQUEST)`。唯一真出口 `GameFlow.self_close(reason)`（emit `pre_self_close` → await 演出钩子 → `SaveSystem.quit_cleanly()`）
- **干净/脏关闭检测**：复用存档系统 `StatsModule.clean_exit` / `had_unclean_exit()`（会话开始即写 clean_exit=false，强杀后残留 → 下次启动读出脏关闭）
- **嘲讽层**：`close_mock.tscn` 常驻 overlay，关闭尝试逐次升级，强杀回来狠话
- **进度**：`AppStateModule` 存 closeai_started/stage/finished
- **对话**：Dialogue Manager + modular_balloon；台词在 `dialogue/closeai_stage{1,2,3}.dialogue`，锚点 start / dirty_return / close_moment
- **开场演出接口**：菜单「开始」后 `GameFlow.register_pre_close_hook()`，演出留给设计接（见 menu.gd `_start_performance`）

## 六、美术与 UI 方向

- 整体：低饱和像素 + 暗调 + 发光元素；UI 金棕玻璃风（玻璃模糊 + 扫描线 + 缺陷边框 + 金棕描边），已统一到菜单/设置/暂停/感谢/气球
- 关卡外壳优先 authored TileMap（碰撞地形 + 背景层分离），不要散落灰盒
- 音效/BGM：环境音 + 简单电子乐；关闭时刻静音处理

## 八、风险提示

- **战斗必须极简**：1 种敌人，接触击退，不加血条以外系统。Day 3 卡住就砍掉战斗，改纯飞行告别也成立
- **文本量控制**：每关 ≤ 20 句
- **Day 4 必须留出来打磨**：完成度和手感 > 关卡数量
- **嘲讽要有度**：关闭嘲讽是亮点，但别让玩家真的卡住——任何时候剧情走到关闭时刻都能正常推进
