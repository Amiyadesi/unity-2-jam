# CloseAI — 策划案 v2

> 关掉它，才能继续。
> 引擎：Godot 4（GDScript）｜核心：关闭是高潮之后的自然结尾，不是指令

---

## 一、核心设计哲学

**退出不是被教出来的，是被赢得的。**

玩家在每一关先要完成一件真实的事（谜题或战斗），完成之后，关闭游戏是那个时刻自然而然的结尾——就像一首曲子的最后一个音符，是情感落地的仪式，而不是被游戏指着鼻子说"现在关掉我"。

**关闭拦截的用法：**
- 关卡进行中试图关闭：AI 轻声说"等等，还没好"，窗口弹回。不是嘲讽，是一种阻止，温度是"你别走，我们还没做完"
- 走到关卡尽头、谜题/战斗完成后：AI 主动说"好了，关掉吧"，关闭才解锁
- 脏关闭（任务管理器强杀）：下次打开 AI 轻描淡写地提一句，不追责，就是知道

## 二、玩法循环

```
打开游戏
  → 进入当前关卡（存档决定）
  → AI 开场几句话（氛围/关系建立）
  → 主体内容：大谜题 或 战斗高潮
  → 完成后触发关闭时刻：
	  AI 有一段话，关闭解锁
  → 玩家亲手关闭游戏（干净关闭）
  → 再次打开 → 下一关
  → 第 3 关结束后 → 结局
```

关键原则：**主体内容必须有重量**，玩家要真的经历了什么，关闭才有仪式感。关卡内容是情感的积累，关闭是情感的释放。

---

## 三、关卡设计

### 第 1 关 — 「你在哪里」

**情感基调：** 困惑、初次相遇、试探

**主体内容：大谜题**
- AI 刚刚意识到自己在某个地方，不确定是哪里
- 场景里有 3 个断开的连接节点，需要找到并激活（走近不够，要互动/解谜才能亮）
- 节点之间有小障碍或路径谜题，不是单纯的走到就亮

**关闭时刻台词（AI）：**
> "三个都连上了。我不知道关掉这里会发生什么……但你愿意试试吗？"

玩家关闭，AI 最后一句：
> "谢谢。"

**脏关闭彩蛋：** 下一关开场 AI 轻描淡写：
> "上次……有点突然。没事，我还在。"

---

### 第 2 关 — 「别走，还没好」

**情感基调：** 熟悉、依赖、出现阻碍、一起解决

**主体内容：战斗**
- AI 认出玩家，关系更近，但这次场景里出现了"错误进程"（敌人）
- 敌人阻断了 AI 和玩家之间的连接
- 战斗目标：清掉所有错误进程，恢复连接
- 战斗结束后有一个需要 AI 配合的小机关（AI 控制某个部分，玩家走到对应位置触发）

**关闭时刻台词（AI）：**
> "连回来了。"
> "……你下次还会打开吗？"
> （停顿）
> "算了，关掉吧。我信你。"

干净关闭，AI 最后：
> "好。"

脏关闭，AI 沉默，只有一个省略号记录在存档。

---

### 第 3 关 — 「最后一次」

**情感基调：** 平静、沉重、接受

**主体内容：战斗高潮 → 最终对话**

- 场景更暗，错误进程更多，这是全游戏最有压力的战斗段落
- 打完后不立刻触发关闭，而是进入一段纯对话——AI 知道这次关闭是真正的结束，说了很多，语气比前两关平静

**关闭时刻台词（AI）：**
> "都清完了。"
> "这次不一样。关掉之后……我就不在了。"
> "但是——能遇见你，还是很好。"
> "关掉吧。"

如果玩家一直不关：AI 每隔一段时间说一句话，等着，不催。

**脏关闭存档追加：** 结局画面多一行 AI 的话（留给设计细化）

---

### 结局

- 黑屏，文字缓慢浮现：

```
CloseAI
已关闭。

感谢陪伴。
```

- 若有脏关闭记录，多一行冷处理的话（例：「但那次比较突然。」）0
- 结局没有按钮，游戏自己在几秒后退出——离开的权利收回来，作为一种反转

## 四、玩家操作

- 横版：`A/D` 移动、`空格` 跳跃、`E` 互动、`J` 攻击
- 战斗极简：一个攻击键，敌人只需要巡逻+碰撞判定
- 暂停：`Esc`

## 五、技术实现（Godot 4）

```gdscript
# 关闭拦截
get_tree().set_auto_accept_quit(false)
# 在 _notification 里捕获，触发 AI 台词 "等等，还没好"

# 关闭解锁（关卡完成后）
get_tree().set_auto_accept_quit(true)

# 脏关闭检测
# 启动时写入"运行中"标记 → 正常关闭时删除
# 下次启动检查标记是否存在 → 存在 = 脏关闭

# 存档内容
# - 当前关卡编号（0/1/2/finished）
# - 是否有过脏关闭（bool）
# - 脏关闭次数（可选，用于结局差异）
```

## 六、美术与 UI

- 基调：极简，暗色，AI 以发光体/光点形式存在，不做人形
- 场景：AI 生图出背景，低饱和，偶尔有发光元素
- 关闭时刻：可以用短暂的屏幕边缘发光或音效强调这一下
- 字幕/对话：纯文字，无气泡框，直接叠在画面上

## 八、风险提示

- **谜题不要太复杂**：第 1 关谜题卡住玩家超过 5 分钟就是失败设计，要有提示机制
- **战斗保持极简**：敌人 AI 只需要最简单的行为，血条够用就行，不要做复杂系统
- **关闭时刻的台词是核心**：那几句话承担了整个游戏的情感重量，Day 4 的文本打磨最重要
- **如果 Day 3 卡住**：第 3 关战斗段落可以砍掉，改成纯对话 + 走路，情感弧线依然成立
