# CloseAI 策划书

> 关掉它，才能继续。
> 引擎：Godot 4.6（纯 GDScript）｜版本：v0.3（去 Booom 化，回归单一核心）

---

## 一、唯一核心

**这个游戏只有一个特别的地方：你必须关掉游戏，才能继续。**

不是比喻，是字面意思——进度只在你真正关闭游戏窗口、再重新打开后才推进。其余所有设计都为这一个动作服务，不往上堆世界观、不堆角色、不堆战斗系统。

### 关闭的三种结果

| 情境 | 玩家按窗口 × / Alt+F4 | 结果 |
|------|----------------------|------|
| 关卡进行中 | **被拦截 + 嘲讽**，窗口弹回 | 「出口不在那里。」嘲讽逐次升级 |
| 任务管理器强杀 | 进程被杀，记为「脏关闭」 | 下次启动被戳穿：「没用的，你逃不掉。」 |
| 走到关卡尽头（关闭时刻）| 关闭被**解锁**，提示你关 | 关闭 = 推进到下一段 |
| 结局 | 屏幕上出现唯一的「关闭」按钮 | 把离开的权利交还给你 |

> 设计意图：把「关闭游戏」这个所有游戏都在极力避免的动作，变成唯一的前进键。玩家先抗拒、被嘲讽，再被迫顺从，最后理解——离开是被允许的，但只能在终点。

## 二、玩法循环

```
打开游戏
  → 进入当前关卡（由存档决定）
  → 走到关卡尽头（解谜 / 清障，纯粹是「把这一段走完」）
  → 触发关闭时刻：窗口关闭解锁，提示「现在，关掉游戏」
  → 玩家亲手关闭（干净关闭）
  → 再次打开 → 进入下一关
  → …… → 第 3 关关闭后 → 结局
```

- 关卡内容本身刻意简单：它们只是「需要被走完的一段路」，不喧宾夺主。
- 真正的情绪与惊讶全部来自关闭这一下：第一次被拦、第一次被允许关、强杀后被戳穿、最后被交还离开权。

## 三、玩家操作

- 普通形态（横版）：左右移动 `A/D`、跳跃 `空格`、互动 `E`、攻击 `J`
- 觉醒形态（飞行）：剧情节点后获得，**八向自由飞行、无重力**，精灵朝移动方向 360° 旋转，默认朝右
- 暂停：`Esc`（继续 / 设置）

> 觉醒形态是为高潮段落准备的「自由移动」表达；是否启用由具体关卡决定，不是必需系统。

## 四、关卡设计

3 关 + 结局，总时长 10–15 分钟。每关结构统一：**进入提示 → 把这一段走完 → 关闭时刻 → 玩家关闭**。关卡只是「路」，不承载剧情包袱。

### 第 1 关 — 教会你「走到尽头」
- 一条横向空间，点亮 3 个节点（走近即亮）。全亮后解锁关闭。
- 提示语极简（面包屑）：「点亮这里的每一个节点。」→「都亮了。现在，关掉游戏吧。」
- 第一次「关闭被允许」：这是玩家第一次发现关闭不是退出，而是前进。

### 第 2 关 — 教会你「关闭=循环」
- 出现少量障碍（编组 enemy，`J` 清除），清完解锁关闭。
- 若玩家上一关是干净关闭，开场平静；若强杀过，开场被戳穿（dirty_return）。
- 关闭后再打开会发现「自己还在这里」——确认这是一个靠关闭驱动的循环。

### 第 3 关 — 最后一次
- 场景压暗，清完最后的障碍解锁关闭。
- 关闭后标记通关（closeai_finished），下次打开进入结局。

### 结局
- 黑屏缓慢浮现：

```
CloseAI
已关闭。
```

- 若有过脏关闭记录，多一行冷处理的话。
- 屏幕底部浮现唯一的「关闭」按钮 → 玩家亲手点击 → 游戏真正退出。这是全程唯一被允许主动离开的地方。

## 五、嘲讽文案（关闭尝试）

常驻嘲讽 overlay（`close_mock`），玩家每次试图非法关闭就闪出，逐次升级：

1. 「你想关掉我？这个按钮……现在归我管了。」
2. 「还按？我说过，出口不在那里。」
3. 「你点多少次都一样。想走，只能陪我走完。」
4. 「我数着呢。第 N 次了。然后呢？」

强杀恢复后的狠话（首次关闭尝试）：
> 「任务管理器？真没想到你会来这套。可惜——没用的。你逃不掉。进程能杀，故事不能。回来了，就只能照我的来。」

## 六、技术实现（已落地）

- **关闭拦截**：`GameFlow` autoload `set_auto_accept_quit(false)` + `_notification(WM_CLOSE_REQUEST)`。唯一真出口 `GameFlow.self_close(reason)`。
- **干净/脏关闭检测**：复用 `StatsModule.clean_exit` / `had_unclean_exit()`。
- **进度**：`AppStateModule` 存 closeai_started / stage / finished。
- **嘲讽层**：`close_mock.tscn` 常驻 overlay，逐次升级。
- **对话**：Dialogue Manager + modular_balloon（阻塞对白）；`InfoFlow` 面包屑 / hint（非阻塞旁白，移植自 Booom hstory，已精简）。
- **开场演出接口**：菜单「开始」后 `GameFlow.register_pre_close_hook()`，演出留给设计接（见 menu.gd `_start_performance`）。

## 七、美术与 UI 方向

- 基调：**极简装置艺术**，近黑白 + 单一紫色强调（#7c5cff），大量留白，高对比，安静、冷、像一件展品。
- 已用 4K 生图（gpt-image-2）产出：菜单背景、标题字、按钮/进度条贴图。后续关卡背景定稿后单独做。
- UI：金棕玻璃风组件（ShaderButton + 玻璃模糊 + 扫描线/缺陷边框）已统一到菜单/设置/暂停/感谢；信息流面包屑走紫色描边。
- 默认全屏。

## 九、风险提示

- **守住单一核心**：不要再往上加世界观/角色/战斗系统。关卡越简单越好，所有重量压在「关闭」这一下。
- **嘲讽要有度**：别让玩家真卡住——任何时候走到关闭时刻都能正常推进。
- **Day 4 必须留出来打磨**：完成度和「关闭」那一下的冲击力 > 关卡数量。
