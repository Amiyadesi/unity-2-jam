# Unity2 Game Jam 提交清单

## 账号和领奖

- 查看报名邮箱，加入邮件里的 QQ 群。
- 登录 Unity2 账号，确认参赛套餐到账。
- 套餐未到账时私信工作人员。

## 仓库

- GitHub 仓库保持 Public。
- README 必须包含：
  - 项目名
  - 这是什么
  - 怎么跑
  - 用了什么 Unity2.Ai API / 模型
  - AI 辅助范围
- `.env` 不提交，`.env.example` 提交。
- 提交频率自然展开，避免最后一天一次性堆完。

## 作品

- 做一个可玩的完整小循环，不追大体量。
- 用 Unity2.Ai API 的地方要能被评委看见，别只做隐藏调用。
- 演示视频讲清楚：
  - 游戏目标
  - API 用在哪里
  - 最有趣的一次玩家决策

## 提交后

- 在专题页投稿，附 GitHub Public 链接。
- 附 README 和演示视频/截图。
- 在群里分享思路、截图或短视频。

## 风险

- 不套壳、不抄袭。
- 记录哪些代码/素材由 AI 辅助生成。
- 使用第三方素材时保留来源和许可证。
